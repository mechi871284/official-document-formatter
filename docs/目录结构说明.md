# 目录结构说明

## 项目目录结构

```
公文格式化工具/
├── src/                          # 源代码目录
│   ├── __init__.py               # Python包初始化文件
│   └── main.py                   # 主程序入口
├── tests/                        # 测试目录
│   ├── __init__.py               # Python包初始化文件
│   └── test_main.py              # 单元测试文件
── assets/                       # 资源文件目录
│   └── icons/                    # 图标资源
│       ├── 公文格式化工具图标设计.png  # 图标源文件（PNG格式）
│       └── 公文格式化工具图标设计.ico  # 生成的图标文件（ICO格式）
── scripts/                      # 工具脚本目录
│   └── convert_icon.py           # 图标转换脚本（PNG转ICO）
├── config/                       # 配置文件目录
│   └── 公文格式化工具.spec       # PyInstaller打包配置文件
├── docs/                         # 文档目录
│   ├── README.md                 # 项目说明文档（根目录）
│   └── 目录结构说明.md           # 本文件
├── build/                        # 构建中间文件（自动生成，已忽略）
├── dist/                         # 构建输出目录（自动生成，已忽略）
│   └── 公文格式化工具.exe        # 最终可执行文件
├── .venv/                        # Python虚拟环境（自动生成，已忽略）
├── .pytest_cache/                # pytest缓存（自动生成，已忽略）
├── __pycache__/                  # Python字节码缓存（自动生成，已忽略）
├── .gitignore                    # Git忽略文件配置
├── requirements.txt              # Python依赖列表
└── README.md                     # 项目说明文档
```

## 目录功能说明

### src/ - 源代码目录

存放项目的所有Python源代码文件。

| 文件 | 说明 |
|------|------|
| `__init__.py` | Python包标识文件 |
| `main.py` | 主程序入口，包含公文格式化的核心逻辑 |

### tests/ - 测试目录

存放项目的单元测试和集成测试文件。

| 文件 | 说明 |
|------|------|
| `__init__.py` | Python包标识文件 |
| `test_main.py` | 主程序的单元测试用例 |

### assets/ - 资源文件目录

存放项目使用的静态资源文件。

#### assets/icons/ - 图标资源

| 文件 | 说明 |
|------|------|
| `公文格式化工具图标设计.png` | 图标源文件，PNG格式，用于生成ICO |
| `公文格式化工具图标设计.ico` | 生成的图标文件，包含多尺寸（16x16至256x256） |

### scripts/ - 工具脚本目录

存放项目开发过程中使用的辅助工具脚本。

| 文件 | 说明 |
|------|------|
| `convert_icon.py` | 图标转换工具，将PNG转换为多尺寸ICO文件 |

### config/ - 配置文件目录

存放项目的配置文件。

| 文件 | 说明 |
|------|------|
| `公文格式化工具.spec` | PyInstaller Windows 打包配置文件，定义 exe 名称、图标、打包选项等 |
| `linux.spec` | PyInstaller Linux 打包配置文件 |

### docs/ - 文档目录

存放项目的相关文档。

| 文件 | 说明 |
|------|------|
| `README.md` | 项目说明文档（位于根目录） |
| `目录结构说明.md` | 本文件，详细说明项目目录结构 |

### build/ - 构建中间文件

PyInstaller打包过程中生成的临时文件目录。

- **状态**：自动生成，已添加到 `.gitignore`
- **内容**：中间编译文件、依赖分析结果等
- **操作**：可安全删除，重新打包时会自动生成

### dist/ - 构建输出目录

PyInstaller打包完成后生成的可执行文件目录。

- **状态**：自动生成，已添加到 `.gitignore`
- **内容**：`公文格式化工具.exe` 最终可执行文件
- **操作**：分发给用户的文件位于此目录

### .venv/ - Python虚拟环境

项目依赖的Python虚拟环境目录。

- **状态**：手动创建，已添加到 `.gitignore`
- **内容**：项目依赖的Python包
- **操作**：通过 `pip install -r requirements.txt` 安装依赖

## 文件命名规范

### 源代码文件

- 使用小写字母和下划线命名
- 示例：`main.py`, `test_main.py`

### 资源文件

- 使用中文描述性命名
- 示例：`公文格式化工具图标设计.png`

### 配置文件

- 使用中文描述性命名 + 标准扩展名
- 示例：`公文格式化工具.spec`

### 文档文件

- 使用中文描述性命名
- 示例：`目录结构说明.md`, `README.md`

## 开发工作流

### 1. 环境准备

```bash
# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境
.venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt
```

### 2. 开发调试

```bash
# 运行主程序
python src/main.py

# 运行测试
pytest tests/ -v
```

### 3. 资源处理

```bash
# 转换图标
python scripts/convert_icon.py
```

### 4. 打包发布

```bash
# 打包为exe
pyinstaller config/公文格式化工具.spec

# 输出文件位于 dist/ 目录
```

## 注意事项

1. **不要手动修改 build/ 和 dist/ 目录**：这些目录由PyInstaller自动管理
2. **提交代码前清理缓存**：删除 `__pycache__/` 和 `.pytest_cache/` 目录
3. **依赖管理**：新增依赖后更新 `requirements.txt` 文件
4. **测试覆盖**：新增功能后添加相应的测试用例
